# Volume Labels

All output label maps follow a fixed integer schema regardless of the segmentation algorithm used. This ensures that downstream tools (automesh, autosim) always receive consistent input.

## Schema

### Homogeneous

| Label | Tissue | Present when |
|-------|--------|--------------|
| 0 | Background | Always |
| 1 | Brain (WM + GM) | Always |
| 3 | CSF | Always |
| 4 | Skull | Always |
| 5 | Membranes | `include_membranes = true` |

### Heterogeneous

| Label | Tissue | Present when |
|-------|--------|--------------|
| 0 | Background | Always |
| 1 | White Matter | Always |
| 2 | Gray Matter | Always |
| 3 | CSF | Always |
| 4 | Skull | Always |
| 5 | Membranes | `include_membranes = true` |

> **Note:** Label `2` (Gray Matter) is only present in heterogeneous mode. In homogeneous mode, both WM and GM are merged into label `1`.

## Skull

The skull is not produced by the segmentation algorithm. It is generated by morphologically dilating the brain mask outward and subtracting the brain and CSF regions. This ensures a consistent skull layer regardless of which algorithm is used.

## Overlap resolution

When two labels occupy the same voxel, the following priority order is applied:

```
Membranes > White Matter = Gray Matter > CSF > Skull
```

This means membranes always win over any other tissue, and brain tissue takes priority over CSF and skull.

## Inspecting the labels

Load the `.npy` file in Python to inspect label distribution:

```python
import numpy as np

data = np.load("combined.npy")
labels, counts = np.unique(data, return_counts=True)
for label, count in zip(labels, counts):
    print(f"Label {label}: {count} voxels ({100 * count / data.size:.1f}%)")
```
